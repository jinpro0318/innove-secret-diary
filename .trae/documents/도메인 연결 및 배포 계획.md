# 도메인 연결 및 배포 계획 (www.inaenote.com)

사용자께서 `www.inaenote.com` 도메인에 현재 프로젝트를 연결하여 배포하기를 원하십니다. Next.js 프로젝트의 배포는 **Vercel**이 가장 호환성이 좋고 간편합니다.

## 배포 프로세스 (Deployment Process)
저는 AI 어시스턴트로서 **코드를 준비하고 가이드**를 드릴 수 있지만, **실제 Vercel 계정에 접속하거나 도메인을 구매/설정하는 행위는 사용자가 직접** 하셔야 합니다.

따라서 다음과 같이 단계를 나누어 진행하겠습니다.

### 1. 배포 준비 (AI Task)
- 프로젝트 코드가 최신 상태로 GitHub에 푸시되었는지 확인합니다. (이전 단계에서 완료됨)
- 배포 시 필요한 환경 변수(`.env`) 목록을 정리해드립니다.

### 2. Vercel 배포 가이드 (User Task Guide)
사용자가 직접 수행해야 할 단계를 상세히 안내해드립니다.
1.  **Vercel 가입 및 연동**: GitHub 계정으로 Vercel에 로그인.
2.  **프로젝트 Import**: `innove-secret-diary` 리포지토리 선택.
3.  **환경 변수 설정**: Supabase URL/Key, Gemini API Key 입력.
4.  **도메인 연결**: Vercel 대시보드 > Settings > Domains 메뉴에서 `www.inaenote.com` 입력 및 DNS 설정(A 레코드/CNAME) 안내.

## 실행 계획
1.  **Check**: `git status`로 모든 변경 사항이 커밋되었는지 확인.
2.  **Guide**: 배포 및 도메인 연결을 위한 상세 가이드 문서를 작성하여 제공.
